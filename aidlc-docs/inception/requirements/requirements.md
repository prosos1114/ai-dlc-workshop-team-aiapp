## 2. 기능 요구사항 (Functional Requirements)

### 2.1 고객용 기능 (Customer Interface)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 태블릿 초기 설정 시 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 테이블 비밀번호는 태블릿을 해당 테이블에 바인딩하기 위한 인증 수단
- 1회 로그인 성공 후 로컬 저장된 정보로 자동 로그인
- 세션 만료 시 저장된 정보로 자동 재인증

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동 (탭 또는 사이드바)
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px 버튼)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 화면
- 주문 확정 시: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트
- 주문 성공 시 장바구니 자동 비우기
- 주문 실패 시 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 이용 완료 처리된 주문 제외

### 2.2 관리자용 기능 (Admin Interface)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한 (brute-force 방지)

#### FR-A02: 관리자 계정 관리
- 매장당 다수의 관리자 계정 지원
- 회원가입 기능 포함
- 매장 식별자 기반 계정 연결

#### FR-A03: 매장 등록
- 별도의 매장 등록 API/관리 화면 제공
- 매장 기본 정보 등록 (매장명, 식별자 등)

#### FR-A04: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료
- 상태 변경 방식: 자동 + 수동 혼합 (주문 접수 시 자동으로 '준비중', 관리자가 '완료' 처리)
- 신규 주문 시각적 강조 (색상, 애니메이션)
- 2초 이내 주문 표시

#### FR-A05: 테이블 관리
- 테이블 초기 설정: 테이블 번호 및 비밀번호 설정
- 주문 삭제: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- 테이블 세션 종료(이용 완료): 확인 팝업 → 주문 내역 과거 이력 이동 → 리셋
- 과거 주문 내역 조회: 시간 역순, 날짜 필터링

#### FR-A06: 메뉴 관리 (MVP 포함)
- 메뉴 CRUD (등록, 조회, 수정, 삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증
- 이미지는 S3 업로드

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- 동시 접속 1000명 이상 지원 (대규모 프랜차이즈)
- SSE 기반 실시간 주문 알림 2초 이내 전달
- API 응답 시간 500ms 이내 (95th percentile)

### NFR-02: 보안
- JWT 토큰 기반 인증 (관리자 16시간, 테이블 세션)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한
- HTTPS 필수 (TLS 1.2+)
- CORS 정책 적용
- Security Extension 전체 적용 (SECURITY-01 ~ SECURITY-15)

### NFR-03: 가용성
- 서비스 가용성 99.5% 이상
- 장애 시 graceful degradation

### NFR-04: 확장성
- 수평 확장 가능한 아키텍처
- 매장 수 증가에 따른 선형 확장

### NFR-05: 유지보수성
- 계층형 아키텍처 (Controller - Service - Repository)
- 코드 커버리지 80% 이상
- API 문서화 (Swagger/OpenAPI)

## 4. 제외 기능 (Out of Scope)

requirements/constraints.md에 정의된 대로:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 매출 리포트, 재고, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달, POS, 소셜, 지도, 번역)

## 5. 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보
- **Admin**: 관리자 계정 (매장별 다수)
- **Table**: 테이블 정보 (매장별)
- **TableSession**: 테이블 세션 (고객 이용 단위)
- **Category**: 메뉴 카테고리
- **Menu**: 메뉴 항목
- **Order**: 주문
- **OrderItem**: 주문 항목
- **OrderHistory**: 과거 주문 이력
