# 테이블오더 서비스 - User Stories

## 구성 방식
- **분류**: Feature-Based (기능 단위) + Persona 그룹화 (사용자 유형별)
- **크기**: 작은 단위 (세부 동작별 1개 스토리)
- **Acceptance Criteria**: 상세 (Given/When/Then + 엣지 케이스 + UI 명세)

---

# 고객 (Customer) 스토리

## Epic 1: 테이블 태블릿 인증 및 세션 관리

### US-C01: 태블릿 초기 설정
**As a** 매장 관리자 (최동현)  
**I want to** 태블릿에 매장 식별자, 테이블 번호, 테이블 비밀번호를 입력하여 초기 설정을 완료할 수 있다  
**So that** 해당 태블릿이 특정 테이블에 바인딩되어 고객이 별도 로그인 없이 사용할 수 있다

**Acceptance Criteria:**
1. Given 태블릿이 초기 상태일 때, When 관리자가 매장 식별자를 입력하면, Then 매장 존재 여부가 검증된다
2. Given 유효한 매장 식별자가 입력되었을 때, When 테이블 번호를 입력하면, Then 해당 매장에 등록된 테이블인지 검증된다
3. Given 유효한 테이블 번호가 입력되었을 때, When 올바른 테이블 비밀번호를 입력하면, Then 인증이 성공하고 로그인 정보가 로컬에 저장된다
4. Given 잘못된 테이블 비밀번호가 입력되었을 때, When 인증을 시도하면, Then 에러 메시지가 표시되고 재시도할 수 있다
5. Given 초기 설정이 완료되었을 때, When 설정 정보가 저장되면, Then 이후 자동 로그인이 활성화된다
6. Given 매장 식별자 입력 필드가 표시될 때, When 화면을 확인하면, Then 입력 필드는 최소 44x44px 터치 영역을 가진다
7. Given 네트워크 오류가 발생했을 때, When 초기 설정을 시도하면, Then 적절한 에러 메시지와 재시도 옵션이 표시된다

**요구사항 추적**: FR-C01

---

### US-C02: 자동 로그인
**As a** 고객 (김민지, 박준호)  
**I want to** 태블릿을 켜면 자동으로 로그인되어 바로 메뉴 화면이 표시된다  
**So that** 별도의 인증 절차 없이 즉시 주문을 시작할 수 있다

**Acceptance Criteria:**
1. Given 초기 설정이 완료된 태블릿일 때, When 태블릿을 켜거나 브라우저를 열면, Then 저장된 정보로 자동 로그인되어 메뉴 화면이 표시된다
2. Given 자동 로그인이 진행 중일 때, When 로딩 상태가 표시되면, Then 로딩 인디케이터가 사용자에게 보인다
3. Given 저장된 인증 정보가 만료되었을 때, When 자동 로그인을 시도하면, Then 저장된 정보로 자동 재인증을 시도한다
4. Given 자동 재인증이 실패했을 때, When 에러가 발생하면, Then 초기 설정 화면으로 이동한다
5. Given 네트워크가 불안정할 때, When 자동 로그인을 시도하면, Then 재시도 로직이 동작하고 실패 시 오프라인 안내 메시지가 표시된다
6. Given 페이지를 새로고침했을 때, When 로그인 상태를 확인하면, Then 세션이 유지되어 메뉴 화면이 다시 표시된다
7. Given 자동 로그인 성공 시, When 메뉴 화면이 로드되면, Then 1초 이내에 메뉴 화면이 완전히 표시된다

**요구사항 추적**: FR-C01

## Epic 2: 메뉴 조회 및 탐색

### US-C03: 카테고리별 메뉴 목록 조회
**As a** 고객 (김민지, 박준호)  
**I want to** 카테고리별로 분류된 메뉴 목록을 볼 수 있다  
**So that** 원하는 종류의 메뉴를 빠르게 찾을 수 있다

**Acceptance Criteria:**
1. Given 메뉴 화면에 진입했을 때, When 화면이 로드되면, Then 첫 번째 카테고리의 메뉴가 기본으로 표시된다
2. Given 카테고리 탭이 표시될 때, When 다른 카테고리를 탭하면, Then 해당 카테고리의 메뉴 목록으로 즉시 전환된다
3. Given 메뉴 목록이 표시될 때, When 각 메뉴 카드를 확인하면, Then 메뉴명, 가격, 이미지가 카드 형태로 표시된다
4. Given 카테고리에 메뉴가 없을 때, When 해당 카테고리를 선택하면, Then "등록된 메뉴가 없습니다" 안내 메시지가 표시된다
5. Given 메뉴 이미지가 로드 실패했을 때, When 메뉴 카드를 확인하면, Then 기본 플레이스홀더 이미지가 표시된다
6. Given 카테고리 탭 영역이 표시될 때, When UI를 확인하면, Then 각 탭은 최소 44x44px 터치 영역을 가진다
7. Given 메뉴 목록이 길 때, When 스크롤하면, Then 부드러운 스크롤이 동작하고 카테고리 탭은 상단에 고정된다
8. Given 메뉴 데이터 로딩 중일 때, When 화면을 확인하면, Then 스켈레톤 UI 또는 로딩 인디케이터가 표시된다

**요구사항 추적**: FR-C02

### US-C04: 메뉴 상세 정보 확인
**As a** 고객 (김민지, 박준호)  
**I want to** 메뉴 카드를 탭하여 상세 정보(설명, 큰 이미지)를 확인할 수 있다  
**So that** 주문 전에 메뉴에 대한 충분한 정보를 얻을 수 있다

**Acceptance Criteria:**
1. Given 메뉴 목록이 표시될 때, When 메뉴 카드를 탭하면, Then 메뉴 상세 정보(메뉴명, 가격, 설명, 큰 이미지)가 표시된다
2. Given 메뉴 상세 화면이 표시될 때, When "장바구니 담기" 버튼을 탭하면, Then 해당 메뉴가 장바구니에 추가된다
3. Given 메뉴 상세 화면이 표시될 때, When "닫기" 또는 뒤로가기를 탭하면, Then 메뉴 목록으로 돌아간다
4. Given 메뉴 설명이 길 때, When 상세 화면을 확인하면, Then 스크롤 가능한 영역에 전체 설명이 표시된다
5. Given 메뉴 상세 화면에서, When 이미지를 확인하면, Then 고해상도 이미지가 표시된다
6. Given 메뉴 상세 화면의 버튼들이 표시될 때, When UI를 확인하면, Then 모든 버튼은 최소 44x44px 터치 영역을 가진다
7. Given 메뉴 상세 화면이 표시될 때, When 가격을 확인하면, Then 가격이 원화(₩) 형식으로 명확하게 표시된다

**요구사항 추적**: FR-C02

## Epic 3: 장바구니 관리

### US-C05: 메뉴를 장바구니에 추가
**As a** 고객 (김민지)  
**I want to** 원하는 메뉴를 장바구니에 추가할 수 있다  
**So that** 여러 메뉴를 모아서 한 번에 주문할 수 있다

**Acceptance Criteria:**
1. Given 메뉴 목록 또는 상세 화면에서, When "장바구니 담기" 버튼을 탭하면, Then 해당 메뉴가 수량 1로 장바구니에 추가된다
2. Given 이미 장바구니에 있는 메뉴를 다시 추가할 때, When "장바구니 담기"를 탭하면, Then 해당 메뉴의 수량이 1 증가한다
3. Given 메뉴가 장바구니에 추가되었을 때, When 추가 완료되면, Then 시각적 피드백(토스트 메시지 또는 애니메이션)이 표시된다
4. Given 장바구니에 메뉴가 있을 때, When 장바구니 아이콘을 확인하면, Then 현재 담긴 메뉴 총 수량이 배지로 표시된다
5. Given 메뉴를 장바구니에 추가한 후, When 페이지를 새로고침하면, Then 장바구니 내용이 유지된다 (로컬 저장)
6. Given 장바구니 담기 버튼이 표시될 때, When UI를 확인하면, Then 버튼은 최소 44x44px 터치 영역을 가지며 명확한 레이블이 있다
7. Given 네트워크 오류 상태에서도, When 장바구니에 메뉴를 추가하면, Then 로컬에 정상적으로 저장된다 (서버 통신 불필요)

**요구사항 추적**: FR-C03

---

### US-C06: 장바구니 수량 조절
**As a** 고객 (김민지, 박준호)  
**I want to** 장바구니에 담긴 메뉴의 수량을 증가/감소시킬 수 있다  
**So that** 원하는 수량으로 정확하게 주문할 수 있다

**Acceptance Criteria:**
1. Given 장바구니에 메뉴가 있을 때, When "+" 버튼을 탭하면, Then 해당 메뉴의 수량이 1 증가한다
2. Given 장바구니에 메뉴가 있을 때, When "-" 버튼을 탭하면, Then 해당 메뉴의 수량이 1 감소한다
3. Given 메뉴 수량이 1일 때, When "-" 버튼을 탭하면, Then 해당 메뉴가 장바구니에서 제거된다
4. Given 수량이 변경될 때, When 총 금액을 확인하면, Then 총 금액이 실시간으로 재계산되어 표시된다
5. Given 수량 조절 버튼이 표시될 때, When UI를 확인하면, Then "+"/"-" 버튼은 최소 44x44px 터치 영역을 가진다
6. Given 수량이 변경된 후, When 페이지를 새로고침하면, Then 변경된 수량이 유지된다
7. Given 수량이 최대값(99)에 도달했을 때, When "+" 버튼을 탭하면, Then 수량이 더 이상 증가하지 않고 시각적 피드백이 표시된다

**요구사항 추적**: FR-C03

---

### US-C07: 장바구니 항목 삭제 및 비우기
**As a** 고객 (김민지, 박준호)  
**I want to** 장바구니에서 특정 메뉴를 삭제하거나 전체를 비울 수 있다  
**So that** 주문 내용을 자유롭게 수정할 수 있다

**Acceptance Criteria:**
1. Given 장바구니에 메뉴가 있을 때, When 특정 메뉴의 "삭제" 버튼을 탭하면, Then 해당 메뉴가 장바구니에서 제거된다
2. Given 장바구니에 여러 메뉴가 있을 때, When "전체 비우기" 버튼을 탭하면, Then 확인 팝업이 표시된다
3. Given 확인 팝업이 표시될 때, When "확인"을 탭하면, Then 장바구니의 모든 메뉴가 제거된다
4. Given 확인 팝업이 표시될 때, When "취소"를 탭하면, Then 장바구니가 그대로 유지된다
5. Given 메뉴가 삭제된 후, When 총 금액을 확인하면, Then 총 금액이 즉시 재계산된다
6. Given 장바구니가 비어있을 때, When 장바구니 화면을 확인하면, Then "장바구니가 비어있습니다" 안내와 메뉴 화면으로 이동 버튼이 표시된다
7. Given 삭제 동작 후, When 페이지를 새로고침하면, Then 삭제된 상태가 유지된다

**요구사항 추적**: FR-C03

### US-C08: 장바구니 총 금액 확인
**As a** 고객 (김민지, 박준호)  
**I want to** 장바구니의 총 주문 금액을 실시간으로 확인할 수 있다  
**So that** 예산 내에서 주문을 관리할 수 있다

**Acceptance Criteria:**
1. Given 장바구니에 메뉴가 있을 때, When 장바구니 화면을 확인하면, Then 각 메뉴의 소계(단가 x 수량)와 총 금액이 표시된다
2. Given 수량이 변경될 때, When 총 금액 영역을 확인하면, Then 즉시 재계산된 금액이 표시된다
3. Given 총 금액이 표시될 때, When 금액 형식을 확인하면, Then 원화(₩) 형식으로 천 단위 구분자와 함께 표시된다
4. Given 장바구니 하단에 총 금액이 표시될 때, When 스크롤해도, Then 총 금액 영역은 하단에 고정되어 항상 보인다
5. Given 메뉴가 추가/삭제될 때, When 총 금액을 확인하면, Then 0.5초 이내에 업데이트된다
6. Given 장바구니가 비어있을 때, When 총 금액을 확인하면, Then ₩0으로 표시된다
7. Given 장바구니 화면에서, When 각 항목을 확인하면, Then 메뉴명, 단가, 수량, 소계가 명확히 구분되어 표시된다

**요구사항 추적**: FR-C03

## Epic 4: 주문 생성

### US-C09: 주문 내역 최종 확인
**As a** 고객 (김민지, 박준호)  
**I want to** 주문 확정 전에 주문 내역을 최종 확인할 수 있다  
**So that** 실수 없이 정확한 주문을 할 수 있다

**Acceptance Criteria:**
1. Given 장바구니에 메뉴가 있을 때, When "주문하기" 버튼을 탭하면, Then 주문 확인 화면이 표시된다
2. Given 주문 확인 화면이 표시될 때, When 내용을 확인하면, Then 모든 메뉴명, 수량, 단가, 소계, 총 금액이 표시된다
3. Given 주문 확인 화면에서, When "주문 확정" 버튼을 탭하면, Then 주문이 서버로 전송된다
4. Given 주문 확인 화면에서, When "뒤로가기"를 탭하면, Then 장바구니 화면으로 돌아간다
5. Given 주문 확인 화면이 표시될 때, When 테이블 정보를 확인하면, Then 현재 테이블 번호가 표시된다
6. Given 장바구니가 비어있을 때, When "주문하기" 버튼을 탭하면, Then 버튼이 비활성화되어 있거나 안내 메시지가 표시된다
7. Given 주문 확인 화면의 "주문 확정" 버튼이 표시될 때, When UI를 확인하면, Then 버튼은 눈에 띄는 색상과 충분한 크기(최소 44px 높이)를 가진다

**요구사항 추적**: FR-C04

### US-C10: 주문 확정 및 성공 처리
**As a** 고객 (김민지, 박준호)  
**I want to** 주문을 확정하고 성공 결과를 확인할 수 있다  
**So that** 주문이 정상적으로 접수되었음을 알 수 있다

**Acceptance Criteria:**
1. Given 주문 확정 버튼을 탭했을 때, When 주문이 성공하면, Then 주문 번호가 포함된 성공 화면이 표시된다
2. Given 주문 성공 화면이 표시될 때, When 5초가 경과하면, Then 자동으로 메뉴 화면으로 리다이렉트된다
3. Given 주문이 성공했을 때, When 장바구니를 확인하면, Then 장바구니가 자동으로 비워져 있다

5. Given 주문 성공 화면에서, When "메뉴로 돌아가기" 버튼을 탭하면, Then 5초 대기 없이 즉시 메뉴 화면으로 이동한다

## Epic 5: 주문 내역 조회

**So that** 이전에 무엇을 주문했는지, 현재 상태가 어떤지 파악할 수 있다

2. Given 주문 목록이 표시될 때, When 각 주문을 확인하면, Then 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다
3. Given 주문 상태가 표시될 때, When 상태를 확인하면, Then 대기중/준비중/완료가 색상으로 구분되어 표시된다

5. Given 현재 세션에 주문이 없을 때, When 주문 내역을 확인하면, Then "주문 내역이 없습니다" 안내가 표시된다
6. Given 주문 목록이 길 때, When 스크롤하면, Then 부드러운 스크롤이 동작한다
7. Given 주문 상태가 변경되었을 때, When 주문 내역 화면을 새로고침하면, Then 최신 상태가 반영된다
8. Given 주문 내역 화면에서, When 특정 주문을 탭하면, Then 해당 주문의 상세 메뉴 목록이 펼쳐진다

## Epic 8: 실시간 주문 모니터링

2. Given 테이블 카드가 표시될 때, When 각 카드를 확인하면, Then 테이블 번호, 총 주문액, 최신 주문 미리보기가 표시된다
3. Given 활성 주문이 있는 테이블과 없는 테이블이 있을 때, When 대시보드를 확인하면, Then 활성 주문이 있는 테이블이 시각적으로 구분된다
4. Given 테이블 카드를 클릭했을 때, When 상세 보기가 열리면, Then 해당 테이블의 전체 주문 목록과 메뉴 상세가 표시된다
5. Given 대시보드가 표시될 때, When 테이블 필터를 사용하면, Then 특정 테이블만 필터링하여 볼 수 있다
6. Given 주문이 없는 테이블이 있을 때, When 카드를 확인하면, Then "주문 없음" 상태로 표시된다
7. Given 대시보드 로딩 중일 때, When 화면을 확인하면, Then 로딩 인디케이터가 표시된다
8. Given 많은 테이블이 있을 때, When 대시보드를 확인하면, Then 반응형 그리드로 화면 크기에 맞게 배치된다

### US-A06: 실시간 신규 주문 수신

5. Given 신규 주문이 수신되었을 때, When 테이블 카드의 총 주문액을 확인하면, Then 즉시 재계산되어 업데이트된다
6. Given 여러 테이블에서 동시에 주문이 들어올 때, When 대시보드를 확인하면, Then 모든 주문이 각 테이블 카드에 정확히 반영된다
7. Given 브라우저 탭이 비활성 상태일 때, When 신규 주문이 들어오면, Then 탭 복귀 시 최신 상태가 반영되어 있다

### US-A07: 주문 상태 변경
**As a** 매장 관리자 (이서연)  
**I want to** 주문 상태를 대기중 → 준비중 → 완료로 변경할 수 있다  
**So that** 주문 처리 진행 상황을 관리하고 고객에게 알릴 수 있다

**Acceptance Criteria:**
1. Given 주문이 '대기중' 상태일 때, When "준비중" 버튼을 클릭하면, Then 주문 상태가 '준비중'으로 변경된다
2. Given 주문이 '준비중' 상태일 때, When "완료" 버튼을 클릭하면, Then 주문 상태가 '완료'로 변경된다
3. Given 주문 상태가 변경되었을 때, When 대시보드를 확인하면, Then 변경된 상태가 즉시 반영된다
4. Given 주문 상태가 변경되었을 때, When 고객 측 주문 내역을 확인하면, Then 변경된 상태가 반영된다
5. Given '완료' 상태의 주문일 때, When 상태 변경 버튼을 확인하면, Then 더 이상 상태 변경 버튼이 표시되지 않는다
6. Given 상태 변경 실패 시, When 에러가 발생하면, Then 에러 메시지가 표시되고 이전 상태가 유지된다
7. Given 상태 변경 버튼이 표시될 때, When UI를 확인하면, Then 각 상태별로 다른 색상의 버튼이 표시된다
8. Given 신규 주문이 접수되었을 때, When 주문 상태를 확인하면, Then 자동으로 '대기중' 상태로 설정된다

**요구사항 추적**: FR-A04

## Epic 9: 테이블 관리

### US-A08: 테이블 초기 설정
**As a** 매장 오너 (최동현)  
**I want to** 테이블 번호와 비밀번호를 설정할 수 있다  
**So that** 태블릿을 해당 테이블에 바인딩할 수 있다

**Acceptance Criteria:**
1. Given 테이블 관리 화면에서, When 새 테이블 추가 버튼을 클릭하면, Then 테이블 번호와 비밀번호 입력 폼이 표시된다
2. Given 테이블 정보를 입력했을 때, When 저장 버튼을 클릭하면, Then 테이블이 생성되고 목록에 추가된다
3. Given 이미 존재하는 테이블 번호를 입력했을 때, When 저장을 시도하면, Then "이미 존재하는 테이블 번호입니다" 에러가 표시된다
4. Given 테이블이 생성되었을 때, When 태블릿에서 해당 정보로 초기 설정을 시도하면, Then 정상적으로 바인딩된다
5. Given 테이블 비밀번호 설정 시, When 비밀번호를 입력하면, Then 최소 4자리 이상이어야 한다
6. Given 테이블 목록이 표시될 때, When 각 테이블을 확인하면, Then 테이블 번호, 현재 세션 상태, 활성 주문 수가 표시된다
7. Given 기존 테이블의 비밀번호를 변경할 때, When 새 비밀번호를 입력하고 저장하면, Then 비밀번호가 업데이트된다

**요구사항 추적**: FR-A05

### US-A09: 주문 삭제 (직권 수정)
**As a** 매장 관리자 (이서연)  
**I want to** 특정 주문을 삭제할 수 있다  
**So that** 잘못된 주문이나 취소 요청을 처리할 수 있다

**Acceptance Criteria:**
1. Given 테이블 상세 보기에서, When 특정 주문의 "삭제" 버튼을 클릭하면, Then 확인 팝업이 표시된다
2. Given 확인 팝업이 표시될 때, When "확인"을 클릭하면, Then 주문이 즉시 삭제된다
3. Given 확인 팝업이 표시될 때, When "취소"를 클릭하면, Then 주문이 유지된다
4. Given 주문이 삭제되었을 때, When 테이블 카드를 확인하면, Then 총 주문액이 재계산되어 표시된다
5. Given 주문이 삭제되었을 때, When 고객 측 주문 내역을 확인하면, Then 삭제된 주문이 목록에서 제거된다
6. Given 주문 삭제 성공 시, When 피드백을 확인하면, Then "주문이 삭제되었습니다" 성공 메시지가 표시된다
7. Given 주문 삭제 실패 시, When 에러가 발생하면, Then 에러 메시지가 표시되고 주문이 유지된다

**요구사항 추적**: FR-A05

### US-A10: 테이블 이용 완료 처리
**As a** 매장 관리자 (이서연)  
**I want to** 테이블 세션을 종료(이용 완료)할 수 있다  
**So that** 새 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다

**Acceptance Criteria:**
1. Given 테이블 카드 또는 상세 보기에서, When "이용 완료" 버튼을 클릭하면, Then 확인 팝업이 표시된다
2. Given 확인 팝업이 표시될 때, When "확인"을 클릭하면, Then 테이블 세션이 종료된다
3. Given 세션이 종료되었을 때, When 해당 테이블의 주문 내역을 확인하면, Then 현재 주문 목록이 비워지고 총 주문액이 0이 된다
4. Given 세션이 종료되었을 때, When 과거 내역을 확인하면, Then 해당 세션의 주문이 과거 이력으로 이동되어 있다
5. Given 세션 종료 후, When 고객 태블릿에서 주문 내역을 확인하면, Then 이전 세션 주문이 표시되지 않는다
6. Given 세션 종료 후, When 새 고객이 주문을 생성하면, Then 새로운 세션이 시작된다
7. Given 이용 완료 성공 시, When 피드백을 확인하면, Then "이용 완료 처리되었습니다" 성공 메시지가 표시된다
8. Given 이용 완료 실패 시, When 에러가 발생하면, Then 에러 메시지가 표시되고 세션이 유지된다

**요구사항 추적**: FR-A05

### US-A11: 과거 주문 내역 조회
**As a** 매장 관리자 (이서연), 매장 오너 (최동현)  
**I want to** 테이블별 과거 주문 내역을 조회할 수 있다  
**So that** 이전 매출 현황을 확인하고 운영에 참고할 수 있다

**Acceptance Criteria:**
1. Given 테이블 상세 보기에서, When "과거 내역" 버튼을 클릭하면, Then 과거 주문 목록이 시간 역순으로 표시된다
2. Given 과거 내역이 표시될 때, When 각 주문을 확인하면, Then 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
3. Given 과거 내역 화면에서, When 날짜 필터를 사용하면, Then 선택한 날짜 범위의 주문만 표시된다
4. Given 과거 내역이 많을 때, When 스크롤하면, Then 페이지네이션 또는 무한 스크롤로 추가 데이터가 로드된다
5. Given 과거 내역 화면에서, When "닫기" 버튼을 클릭하면, Then 대시보드로 복귀한다
6. Given 과거 내역이 없을 때, When 화면을 확인하면, Then "과거 주문 내역이 없습니다" 안내가 표시된다
7. Given 날짜 필터 사용 시, When 시작일이 종료일보다 늦을 때, Then 검증 에러가 표시된다

## Epic 10: 메뉴 관리

### US-A12: 메뉴 등록
**As a** 매장 관리자 (이서연), 매장 오너 (최동현)  
**I want to** 새로운 메뉴를 등록할 수 있다  
**So that** 고객에게 새 메뉴를 제공할 수 있다

**Acceptance Criteria:**
1. Given 메뉴 관리 화면에서, When "메뉴 추가" 버튼을 클릭하면, Then 메뉴 등록 폼이 표시된다
2. Given 메뉴 등록 폼에서, When 메뉴명, 가격, 설명, 카테고리를 입력하고 저장하면, Then 메뉴가 생성된다
3. Given 이미지 업로드 시, When 이미지 파일을 선택하면, Then S3에 업로드되고 URL이 메뉴에 연결된다
4. Given 필수 필드(메뉴명, 가격, 카테고리)가 비어있을 때, When 저장을 시도하면, Then 검증 에러가 표시된다
5. Given 가격 입력 시, When 0 이하 또는 비정상 값을 입력하면, Then "유효한 가격을 입력해주세요" 에러가 표시된다
6. Given 메뉴 등록 성공 시, When 메뉴 목록을 확인하면, Then 새 메뉴가 목록에 추가되어 있다
7. Given 이미지 업로드 실패 시, When 에러가 발생하면, Then 에러 메시지가 표시되고 이미지 없이 저장할 수 있는 옵션이 제공된다
8. Given 메뉴 등록 시, When 카테고리를 선택하면, Then 기존 카테고리 목록에서 선택하거나 새 카테고리를 생성할 수 있다

**요구사항 추적**: FR-A06

### US-A13: 메뉴 조회 (관리자)
**As a** 매장 관리자 (이서연)  
**I want to** 카테고리별로 등록된 메뉴를 조회할 수 있다  
**So that** 현재 메뉴 현황을 파악하고 관리할 수 있다

**Acceptance Criteria:**
1. Given 메뉴 관리 화면에 진입했을 때, When 화면이 로드되면, Then 전체 메뉴가 카테고리별로 그룹화되어 표시된다
2. Given 카테고리 필터가 표시될 때, When 특정 카테고리를 선택하면, Then 해당 카테고리의 메뉴만 표시된다
3. Given 메뉴 목록이 표시될 때, When 각 메뉴를 확인하면, Then 메뉴명, 가격, 카테고리, 이미지 썸네일, 노출 순서가 표시된다
4. Given 메뉴가 없을 때, When 화면을 확인하면, Then "등록된 메뉴가 없습니다" 안내와 메뉴 추가 버튼이 표시된다
5. Given 메뉴 목록이 표시될 때, When 각 메뉴 항목을 확인하면, Then 수정/삭제 버튼이 함께 표시된다
6. Given 메뉴 목록 로딩 중일 때, When 화면을 확인하면, Then 로딩 인디케이터가 표시된다
7. Given 많은 메뉴가 있을 때, When 목록을 확인하면, Then 스크롤 가능한 목록으로 표시된다

**요구사항 추적**: FR-A06

### US-A14: 메뉴 수정
**As a** 매장 관리자 (이서연)  
**I want to** 기존 메뉴의 정보를 수정할 수 있다  
**So that** 가격 변경, 설명 업데이트 등을 반영할 수 있다

**Acceptance Criteria:**
1. Given 메뉴 목록에서, When 특정 메뉴의 "수정" 버튼을 클릭하면, Then 해당 메뉴의 정보가 채워진 수정 폼이 표시된다
2. Given 수정 폼에서, When 정보를 변경하고 저장하면, Then 메뉴 정보가 업데이트된다
3. Given 수정 폼에서, When "취소"를 클릭하면, Then 변경사항이 저장되지 않고 목록으로 돌아간다
4. Given 이미지를 변경할 때, When 새 이미지를 업로드하면, Then 기존 이미지가 새 이미지로 교체된다
5. Given 필수 필드를 비웠을 때, When 저장을 시도하면, Then 검증 에러가 표시된다
6. Given 메뉴 수정 성공 시, When 고객 화면을 확인하면, Then 변경된 정보가 반영된다
7. Given 수정 중 네트워크 오류 시, When 에러가 발생하면, Then 에러 메시지가 표시되고 입력 내용이 유지된다

**요구사항 추적**: FR-A06

### US-A15: 메뉴 삭제
**As a** 매장 관리자 (이서연), 매장 오너 (최동현)  
**I want to** 더 이상 제공하지 않는 메뉴를 삭제할 수 있다  
**So that** 고객에게 정확한 메뉴만 표시할 수 있다

**Acceptance Criteria:**
1. Given 메뉴 목록에서, When 특정 메뉴의 "삭제" 버튼을 클릭하면, Then 확인 팝업이 표시된다
2. Given 확인 팝업이 표시될 때, When "확인"을 클릭하면, Then 메뉴가 삭제된다
3. Given 확인 팝업이 표시될 때, When "취소"를 클릭하면, Then 메뉴가 유지된다
4. Given 메뉴 삭제 성공 시, When 메뉴 목록을 확인하면, Then 삭제된 메뉴가 목록에서 제거된다
5. Given 메뉴 삭제 성공 시, When 고객 화면을 확인하면, Then 삭제된 메뉴가 표시되지 않는다
6. Given 삭제 실패 시, When 에러가 발생하면, Then 에러 메시지가 표시되고 메뉴가 유지된다
7. Given 현재 주문에 포함된 메뉴를 삭제할 때, When 삭제를 시도하면, Then 경고 메시지가 표시되고 삭제 여부를 재확인한다

**요구사항 추적**: FR-A06

### US-A16: 메뉴 노출 순서 조정
**As a** 매장 관리자 (이서연)  
**I want to** 메뉴의 노출 순서를 조정할 수 있다  
**So that** 인기 메뉴나 추천 메뉴를 상위에 배치할 수 있다

**Acceptance Criteria:**
1. Given 메뉴 관리 화면에서, When 드래그 앤 드롭으로 메뉴 순서를 변경하면, Then 순서가 즉시 반영된다
2. Given 순서가 변경되었을 때, When 저장 버튼을 클릭하면, Then 변경된 순서가 서버에 저장된다
3. Given 순서 변경 후, When 고객 화면을 확인하면, Then 변경된 순서로 메뉴가 표시된다
4. Given 순서 변경 중, When "취소"를 클릭하면, Then 이전 순서로 복원된다
5. Given 같은 카테고리 내에서, When 순서를 변경하면, Then 해당 카테고리 내에서만 순서가 조정된다
6. Given 순서 저장 실패 시, When 에러가 발생하면, Then 에러 메시지가 표시되고 이전 순서가 유지된다
7. Given 터치 디바이스에서, When 순서를 변경하려면, Then 위/아래 화살표 버튼으로도 순서 변경이 가능하다

**요구사항 추적**: FR-A06
