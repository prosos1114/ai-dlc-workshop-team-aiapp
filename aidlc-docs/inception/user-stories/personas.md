## Persona 1: 김민지 (고객 - Customer)

### 기본 정보
| 항목 | 내용 |
|---|---|
| **이름** | 김민지 |
| **나이** | 28세 |
| **직업** | IT 회사 마케팅 팀장 |
| **기술 수준** | 높음 (스마트폰/태블릿 능숙) |

### 배경 스토리
김민지는 바쁜 직장인으로, 점심시간에 동료들과 자주 외식을 합니다. 대기 시간을 싫어하고, 빠르고 효율적인 서비스를 선호합니다. 디지털 기기에 익숙하며, 키오스크나 모바일 주문에 거부감이 없습니다.

### 목표
- 최소한의 시간으로 주문 완료
- 메뉴를 충분히 탐색한 후 결정
- 주문 상태를 실시간으로 확인
- 추가 주문을 편리하게 진행

### 핵심 니즈
- 직관적이고 빠른 주문 프로세스
- 메뉴 정보(이미지, 설명, 가격)의 명확한 표시
- 장바구니에서 쉽게 수량 조절
- 주문 후 상태 확인

### 행동 패턴
- 카테고리를 빠르게 훑어보며 메뉴 선택
- 이미지와 가격을 우선적으로 확인
- 한 번에 여러 메뉴를 장바구니에 담은 후 한꺼번에 주문
- 식사 중 추가 주문 빈번

### 불만 사항
- 직원을 불러야 하는 번거로움
- 주문 후 얼마나 기다려야 하는지 모르는 불안감
- 복잡하거나 느린 인터페이스
- 메뉴 정보가 부족한 경우

### 동기 부여 요소
- 빠른 응답과 즉각적인 피드백
- 깔끔하고 현대적인 UI
- 주문 과정의 자율성과 통제감

## Persona 2: 박준호 (고객 - Customer, 디지털 취약 계층)

### 기본 정보
| 항목 | 내용 |
|---|---|
| **이름** | 박준호 |
| **나이** | 55세 |
| **직업** | 자영업자 |
| **기술 수준** | 낮음 (스마트폰 기본 기능만 사용) |

### 배경 스토리
박준호는 가족 모임이나 지인과의 식사 자리에서 테이블오더를 접합니다. 디지털 기기에 익숙하지 않아 처음에는 당황하지만, 직관적인 인터페이스라면 사용할 의향이 있습니다.

### 목표
- 어렵지 않게 메뉴를 찾고 주문
- 실수 없이 원하는 메뉴를 정확히 주문
- 주문이 제대로 들어갔는지 확인

### 핵심 니즈
- 큰 글씨와 명확한 버튼
- 단순하고 예측 가능한 흐름
- 실수 시 쉽게 수정 가능
- 주문 확인 단계에서 최종 검토

### 행동 패턴
- 메뉴를 천천히 하나씩 살펴봄
- 이미지에 크게 의존하여 메뉴 선택
- 한 번에 하나씩 장바구니에 추가
- 주문 전 여러 번 확인

### 불만 사항
- 너무 작은 버튼이나 글씨
- 복잡한 네비게이션
- 실수했을 때 되돌리기 어려운 경우
- 확인 없이 바로 주문되는 경우

### 동기 부여 요소
- 성공적으로 주문을 완료했을 때의 성취감
- 직원을 부르지 않아도 되는 편리함
- 가족/지인에게 도움을 요청하지 않아도 되는 자립감

## Persona 3: 이서연 (매장 관리자 - Store Admin)

### 기본 정보
| 항목 | 내용 |
|---|---|
| **이름** | 이서연 |
| **나이** | 35세 |
| **직업** | 프랜차이즈 카페 매장 매니저 |
| **기술 수준** | 중간 (POS 시스템 사용 경험, 기본 PC 활용) |

### 배경 스토리
이서연은 프랜차이즈 카페의 매장 매니저로, 매일 수십 건의 주문을 관리합니다. 피크 타임에는 동시에 여러 테이블의 주문을 처리해야 하며, 효율적인 주문 관리 도구가 필수적입니다. 메뉴 변경이나 프로모션 시 빠르게 메뉴를 업데이트해야 합니다.

### 목표
- 모든 테이블의 주문 상황을 한눈에 파악
- 신규 주문을 놓치지 않고 즉시 처리
- 메뉴를 빠르게 등록/수정
- 테이블 회전율을 효율적으로 관리

### 핵심 니즈
- 실시간 주문 알림 및 대시보드
- 직관적인 주문 상태 관리
- 간편한 메뉴 CRUD
- 테이블별 세션 관리 (이용 완료 처리)
- 과거 주문 내역 조회

### 행동 패턴
- 대시보드를 항상 열어두고 모니터링
- 신규 주문 확인 즉시 '준비중'으로 상태 변경
- 음식 완성 시 '완료'로 상태 변경
- 고객 퇴장 시 테이블 이용 완료 처리
- 주기적으로 메뉴 정보 업데이트

### 불만 사항
- 주문을 놓쳐서 고객 불만이 발생하는 경우
- 복잡한 관리 화면으로 인한 조작 실수
- 테이블 상태 파악이 어려운 경우
- 메뉴 수정이 번거로운 경우

### 동기 부여 요소
- 매장 운영 효율성 향상
- 고객 만족도 증가
- 인건비 절감 효과
- 실수 없는 주문 처리

## Persona 4: 최동현 (매장 오너 - Store Owner/Admin)

### 기본 정보
| 항목 | 내용 |
|---|---|
| **이름** | 최동현 |
| **나이** | 42세 |
| **직업** | 음식점 3개 매장 운영 사장 |
| **기술 수준** | 중상 (비즈니스 소프트웨어 활용 능숙) |

### 배경 스토리
최동현은 여러 매장을 운영하는 사장으로, 각 매장의 운영 상황을 효율적으로 관리하고 싶어합니다. 매장 등록, 관리자 계정 생성, 테이블 설정 등 초기 세팅 작업을 직접 수행합니다.

### 목표
- 새 매장을 시스템에 빠르게 등록
- 직원(관리자) 계정을 효율적으로 관리
- 테이블 초기 설정을 간편하게 완료
- 매장별 주문 현황 파악

### 핵심 니즈
- 매장 등록 및 설정 기능
- 관리자 계정 생성/관리
- 테이블 설정 (번호, 비밀번호)
- 과거 주문 내역 조회 및 날짜 필터링

### 행동 패턴
- 새 매장 오픈 시 시스템 초기 설정
- 직원 입/퇴사 시 계정 관리
- 주기적으로 매장 운영 현황 확인
- 문제 발생 시 주문 삭제(직권 수정) 처리

### 불만 사항
- 초기 설정이 복잡한 경우
- 매장별 관리가 분리되지 않는 경우
- 직원 계정 관리가 번거로운 경우

### 동기 부여 요소
- 매장 운영의 디지털화
- 운영 비용 절감
- 체계적인 매장 관리

## 페르소나 간 관계 및 상호작용

```
+------------------+          주문 생성          +------------------+
|                  | --------------------------> |                  |
|  고객 (김민지,   |                             |  매장 관리자      |
|  박준호)         | <-------------------------- |  (이서연)         |
|                  |     주문 상태 업데이트        |                  |
+------------------+                             +------------------+
                                                        |
                                                        | 보고/관리
                                                        v
                                                 +------------------+
                                                 |  매장 오너        |
                                                 |  (최동현)         |
                                                 +------------------+
```

### 상호작용 흐름
1. **고객 → 시스템**: 메뉴 조회, 장바구니 관리, 주문 생성, 주문 내역 조회
2. **시스템 → 관리자**: 실시간 주문 알림 (SSE)
3. **관리자 → 시스템**: 주문 상태 변경, 테이블 관리, 메뉴 관리
4. **시스템 → 고객**: 주문 상태 업데이트
5. **오너 → 시스템**: 매장 등록, 계정 관리, 테이블 초기 설정, 과거 내역 조회
