# Business Logic Model - Unit 4: Customer Frontend

## 1. 전체 애플리케이션 흐름

```
앱 시작
  → 로컬 저장소에 인증 정보 존재?
    → YES: 자동 로그인 시도
      → 성공: 메뉴 화면 (홈)
      → 실패: 초기 설정 화면
    → NO: 초기 설정 화면
      → QR 스캔 → 비밀번호 입력 → 인증 성공
      → 메뉴 화면 (홈)

메뉴 화면 (홈)
  ├── 사이드바: 카테고리 목록 (좌측 고정)
  ├── 메인 영역: 3열 그리드 메뉴 카드
  │   └── 카드 탭 → 확장 카드 (상세 + 장바구니 담기)
  ├── 장바구니 슬라이드 패널 (우측)
  │   └── 3초 무활동 시 자동 닫기
  └── 하단/상단: 주문내역 접근

주문 흐름
  메뉴 담기 → 장바구니 확인 → 주문 확인 → 주문 확정
  → 성공: 주문 번호 표시 → 5초 후 메뉴 화면
  → 실패: 에러 표시 → 재시도 가능

주문 내역
  → SSE로 실시간 상태 업데이트 수신
  → 현재 세션 주문만 표시
```

## 2. 인증 및 세션 관리 로직

### 2.1 QR 코드 초기 설정 (US-C01)
```
QR 스캔 시작 (매장당 QR 코드 1개)
  → 카메라 권한 요청
  → QR 코드 디코딩
    → QR 데이터: { storeCode, totalTables }  ← 매장 코드 + 총 테이블 수
  → 테이블 번호 자동 배정 요청
    POST /api/table/auth/login
    Body: { storeCode, password }
    → 서버가 스캔 순서대로 테이블 번호 배정 (1번부터 totalTables까지)
    → 이미 모든 테이블이 배정된 경우 에러 반환
  → 비밀번호 입력 화면 표시
    → 관리자가 테이블 비밀번호 입력
  → 인증 API 호출
  → 성공:
    → 응답: { token, storeId, tableId, tableNumber }
    → localStorage에 저장: token, storeId, tableId, storeCode, tableNumber
    → 배정된 테이블 번호 확인 화면 표시 ("테이블 3번으로 설정되었습니다")
    → 메뉴 화면으로 이동
  → 실패:
    → 에러 유형별 메시지:
      - 잘못된 비밀번호: "비밀번호가 올바르지 않습니다"
      - 테이블 모두 배정됨: "모든 테이블이 이미 배정되었습니다. 관리자에게 문의하세요"
      - 매장 없음: "존재하지 않는 매장입니다"
    → 재시도 가능
```

### 2.2 자동 로그인 (US-C02)
```
앱 시작 시
  → localStorage에서 인증 정보 확인
    → token, storeId, tableId 존재?
  → YES:
    → 토큰으로 API 호출 시도 (메뉴 조회 등)
    → 성공: 메뉴 화면 표시 (1초 이내)
    → 401 응답:
      → 저장된 정보로 재인증 시도
        POST /api/table/auth/login
        Body: { storeCode, tableNumber, password: (저장된 값) }
      → 성공: 새 토큰 저장 → 메뉴 화면
      → 실패: localStorage 클리어 → 초기 설정 화면
  → NO:
    → 초기 설정 화면 표시
```

**참고**: 테이블 토큰은 365일 유효하므로 자동 재인증이 필요한 경우는 드묾. 비밀번호는 매장 공통 비밀번호로, 초기 설정 시에만 사용. 테이블 번호는 서버가 스캔 순서대로 자동 배정.

## 3. 메뉴 조회 로직

### 3.1 카테고리별 메뉴 조회 (US-C03)
```
메뉴 화면 진입
  → 카테고리 목록 조회
    GET /api/stores/{storeId}/categories
  → 첫 번째 카테고리 자동 선택
  → 해당 카테고리 메뉴 조회
    GET /api/stores/{storeId}/menus?categoryId={categoryId}
  → 3열 그리드로 메뉴 카드 렌더링

카테고리 전환
  → 사이드바에서 카테고리 탭
  → 해당 카테고리 메뉴 조회 (캐싱 적용)
  → 메뉴 목록 즉시 전환
```

### 3.2 메뉴 상세 (US-C04) - 확장 카드
```
메뉴 카드 탭
  → 카드가 제자리에서 확장
  → 확장 영역에 표시:
    - 큰 이미지
    - 메뉴명, 가격 (₩ 형식)
    - 상세 설명
    - "장바구니 담기" 버튼
  → 다른 카드 탭 또는 빈 영역 탭 시 축소

메뉴 상세 조회 API (필요 시)
  GET /api/stores/{storeId}/menus/{menuId}
  → 이미 목록에서 충분한 정보가 있으면 API 호출 생략
```

## 4. 장바구니 관리 로직

### 4.1 장바구니 데이터 구조
```typescript
interface CartItem {
  menuId: number;
  menuName: string;
  unitPrice: number;
  quantity: number;    // 1 ~ 99
  imageUrl?: string;
}

// localStorage key: 'cart_{storeId}_{tableId}'
// 서버 통신 없이 로컬에서만 관리
```

### 4.2 장바구니 추가 (US-C05)
```
"장바구니 담기" 탭
  → 이미 존재하는 메뉴?
    → YES: quantity + 1
    → NO: 새 CartItem 추가 (quantity: 1)
  → localStorage 저장
  → 토스트 메시지 표시 ("장바구니에 추가되었습니다")
  → 장바구니 배지 수량 업데이트
  → 장바구니 슬라이드 패널 자동 열기
```

### 4.3 수량 조절 (US-C06)
```
"+" 버튼 탭
  → quantity < 99?
    → YES: quantity + 1
    → NO: 시각적 피드백 ("최대 수량입니다")
  → 총 금액 재계산
  → localStorage 저장

"-" 버튼 탭
  → quantity > 1?
    → YES: quantity - 1
    → NO (quantity == 1): 해당 항목 제거
  → 총 금액 재계산
  → localStorage 저장
```

### 4.4 장바구니 슬라이드 패널 (Q2 답변 반영)
```
패널 열기 트리거:
  - 장바구니 아이콘 탭
  - 메뉴 장바구니 담기 시 자동 열기

패널 닫기 트리거:
  - 닫기(X) 버튼 탭
  - 3초간 터치/마우스 활동 없으면 자동 닫기
  - 패널 외부 영역 탭

자동 닫기 로직:
  → 패널 열림 시 3초 타이머 시작
  → 패널 내 터치/마우스 이벤트 발생 시 타이머 리셋
  → 타이머 만료 시 슬라이드 애니메이션으로 닫기
  → 패널 닫힐 때 타이머 클리어
```

### 4.5 장바구니 비우기 (US-C07)
```
"전체 비우기" 탭
  → 확인 팝업 표시 ("장바구니를 비우시겠습니까?")
  → "확인" 탭: 모든 항목 제거 → localStorage 업데이트
  → "취소" 탭: 팝업 닫기

개별 삭제
  → 항목의 "삭제" 버튼 탭
  → 해당 항목 즉시 제거 (확인 팝업 없음)
  → 총 금액 재계산
```

## 5. 주문 생성 로직

### 5.1 주문 확인 (US-C09)
```
장바구니에서 "주문하기" 탭
  → 장바구니 비어있으면 버튼 비활성화
  → 주문 확인 화면 표시:
    - 테이블 번호
    - 주문 항목 목록 (메뉴명, 수량, 단가, 소계)
    - 총 금액
    - "주문 확정" 버튼
    - "뒤로가기" 버튼
```

### 5.2 주문 확정 (US-C10)
```
"주문 확정" 탭
  → 버튼 비활성화 (중복 클릭 방지)
  → "주문 처리 중..." 로딩 표시
  → API 호출:
    POST /api/stores/{storeId}/tables/{tableId}/orders
    Body: {
      items: [{ menuId, quantity, menuName, unitPrice }]
    }
  → 성공:
    → 장바구니 비우기 (localStorage 삭제)
    → 주문 성공 화면 표시:
      - 주문 번호
      - 주문 시각
      - 총 금액
      - "메뉴로 돌아가기" 버튼
    → 5초 후 자동으로 메뉴 화면 리다이렉트
    → "메뉴로 돌아가기" 탭 시 즉시 이동
```

### 5.3 주문 실패 처리 (US-C11)
```
API 호출 실패
  → 장바구니 유지 (삭제하지 않음)
  → 에러 유형별 메시지:
    - 네트워크 오류: "네트워크 연결을 확인해주세요"
    - 서버 오류 (5xx): "잠시 후 다시 시도해주세요"
    - 기타: "주문 처리 중 오류가 발생했습니다"
  → "다시 시도" 버튼 표시
  → 실패 횟수 카운트:
    → 3회 이상 연속 실패: "직원에게 문의해주세요" 추가 안내
  → 실패 카운터는 성공 시 리셋
```

## 6. 주문 내역 조회 로직

### 6.1 주문 목록 조회 (US-C12)
```
주문 내역 화면 진입
  → 현재 세션 주문 조회:
    GET /api/stores/{storeId}/tables/{tableId}/orders
  → 시간 순 정렬 (최신 먼저)
  → 각 주문 표시:
    - 주문 번호
    - 주문 시각
    - 메뉴/수량 요약
    - 총 금액
    - 상태 배지 (대기중: 노란색, 준비중: 파란색, 완료: 초록색)
  → 주문 탭 시 상세 메뉴 목록 펼침/접힘

주문 없을 때
  → "주문 내역이 없습니다" 안내 표시
```

### 6.2 SSE 실시간 상태 업데이트 (Q6 답변 반영)
```
주문 내역 화면 진입 시
  → SSE 연결 시작:
    GET /api/stores/{storeId}/orders/stream
  → 이벤트 수신:
    - ORDER_STATUS_CHANGED: 해당 주문 상태 업데이트
    - ORDER_DELETED: 해당 주문 목록에서 제거
  → 화면 이탈 시 SSE 연결 해제

SSE 연결 관리
  → 연결 끊김 시 자동 재연결 (3초 간격, 최대 5회)
  → 재연결 성공 시 전체 주문 목록 재조회 (누락 방지)
  → 연결 상태 인디케이터 표시 (연결됨/재연결 중)
```

## 7. 레이아웃 구조 (가로 모드)

```
+------------------------------------------------------------------+
| [로고]              테이블 5번              [장바구니 아이콘(3)] |  ← 헤더
+----------+-------------------------------------------+-----------+
|          |                                           |           |
| 카테고리  |        3열 메뉴 카드 그리드                | 장바구니   |
| 사이드바  |                                           | 슬라이드   |
|          |  [카드] [카드] [카드]                      | 패널      |
| [음료]   |  [카드] [카드] [카드]                      | (접힘/    |
| [식사]   |  [카드] [카드] [카드]                      |  펼침)    |
| [디저트] |  [카드] [카드] [카드]                      |           |
| [사이드] |                                           |           |
|          |                                           |           |
+----------+-------------------------------------------+-----------+
|  [메뉴]              [주문내역]              [주문하기 ₩25,000] |  ← 하단 바
+------------------------------------------------------------------+
```

- **가로 모드(Landscape) 고정**: 1024x768 이상 기준
- **사이드바**: 좌측 고정, 카테고리 목록
- **메인 영역**: 3열 그리드 메뉴 카드
- **장바구니 패널**: 우측 슬라이드, 3초 자동 닫기
- **하단 바**: 네비게이션 + 주문 총액 표시
