# Business Rules - Unit 4: Customer Frontend

## 1. 인증 규칙

### BR-AUTH-01: QR 코드 초기 설정
- QR 코드는 매장당 1개이며, `storeCode`와 `totalTables`(총 테이블 수)를 포함
- QR 디코딩 실패 시 "유효하지 않은 QR 코드입니다" 메시지 표시
- 카메라 권한 거부 시 수동 입력 폴백 제공 (매장 코드 + 총 테이블 수 직접 입력)
- QR 스캔 후 관리자가 비밀번호만 입력
- 테이블 번호는 서버가 스캔 순서대로 자동 배정 (1번부터 totalTables까지)
- 모든 테이블이 이미 배정된 경우 "모든 테이블이 이미 배정되었습니다" 에러 표시
- 배정 성공 시 배정된 테이블 번호 확인 화면 표시
- 비밀번호는 최소 4자리 이상

### BR-AUTH-02: 자동 로그인
- localStorage에 token, storeId, tableId가 모두 존재해야 자동 로그인 시도
- 토큰 만료(401 응답) 시 저장된 정보로 자동 재인증 1회 시도
- 재인증 실패 시 모든 저장 정보 삭제 후 초기 설정 화면으로 이동
- 자동 로그인 중 로딩 인디케이터 표시 필수

### BR-AUTH-03: 인증 정보 저장
- 저장 위치: localStorage
- 저장 항목: token, storeId, tableId, storeCode, tableNumber, password(암호화)
- 비밀번호는 클라이언트 측 암호화 후 저장 (재인증용)
- 페이지 새로고침 시 세션 유지

## 2. 메뉴 조회 규칙

### BR-MENU-01: 카테고리 표시
- 카테고리는 displayOrder 순으로 정렬
- 첫 번째 카테고리가 기본 선택
- 카테고리에 메뉴가 없으면 "등록된 메뉴가 없습니다" 표시
- 카테고리 전환 시 스크롤 위치 최상단으로 리셋

### BR-MENU-02: 메뉴 카드 표시
- 3열 그리드 레이아웃 (가로 모드 기준)
- 카드 표시 정보: 이미지 썸네일, 메뉴명, 가격
- 이미지 로드 실패 시 기본 플레이스홀더 이미지 표시
- 메뉴는 displayOrder 순으로 정렬
- 가격은 원화(₩) 형식, 천 단위 구분자 포함 (예: ₩12,000)

### BR-MENU-03: 확장 카드 상세
- 카드 탭 시 제자리에서 확장 (다른 카드는 밀려남)
- 확장 시 표시: 큰 이미지, 메뉴명, 가격, 상세 설명, "장바구니 담기" 버튼
- 이미 확장된 카드가 있으면 기존 카드 축소 후 새 카드 확장
- 확장된 카드 재탭 또는 빈 영역 탭 시 축소
- 확장/축소 애니메이션 적용 (300ms)

### BR-MENU-04: 데이터 로딩
- 메뉴 데이터 로딩 중 스켈레톤 UI 표시
- 카테고리별 메뉴 데이터 클라이언트 캐싱 (동일 세션 내)
- 네트워크 오류 시 "메뉴를 불러올 수 없습니다" + 재시도 버튼

## 3. 장바구니 규칙

### BR-CART-01: 데이터 관리
- 장바구니는 localStorage에 저장 (서버 통신 없음)
- 저장 키: `cart_{storeId}_{tableId}`
- 페이지 새로고침 시 장바구니 유지
- 네트워크 오류 상태에서도 장바구니 조작 가능

### BR-CART-02: 수량 제한
- 최소 수량: 1
- 최대 수량: 99
- 수량 1에서 "-" 탭 시 해당 항목 제거
- 수량 99에서 "+" 탭 시 "최대 수량입니다" 피드백

### BR-CART-03: 금액 계산
- 소계 = 단가 x 수량
- 총 금액 = 모든 항목의 소계 합
- 금액 변경 시 0.5초 이내 업데이트
- 금액 형식: ₩ + 천 단위 구분자 (예: ₩25,000)
- 장바구니 비어있을 때 총 금액: ₩0

### BR-CART-04: 슬라이드 패널
- 우측에서 슬라이드 인/아웃 애니메이션
- 열기: 장바구니 아이콘 탭 또는 메뉴 담기 시 자동
- 닫기: X 버튼, 외부 영역 탭, 3초 무활동 자동 닫기
- 자동 닫기 타이머: 패널 내 터치/마우스 이벤트 시 리셋
- 패널 너비: 화면의 약 30% (가로 모드 기준)

### BR-CART-05: 장바구니 배지
- 헤더의 장바구니 아이콘에 총 수량 배지 표시
- 수량 0이면 배지 숨김
- 수량 변경 시 즉시 업데이트

### BR-CART-06: 전체 비우기
- "전체 비우기" 시 확인 팝업 필수
- 개별 항목 삭제는 확인 팝업 없이 즉시 삭제
- 삭제 후 총 금액 즉시 재계산

## 4. 주문 규칙

### BR-ORDER-01: 주문 생성 조건
- 장바구니에 1개 이상의 항목이 있어야 주문 가능
- 장바구니 비어있으면 "주문하기" 버튼 비활성화
- 주문 확정 시 중복 클릭 방지 (버튼 비활성화 + 로딩 표시)

### BR-ORDER-02: 주문 확인 화면
- 표시 정보: 테이블 번호, 주문 항목 목록, 총 금액
- 각 항목: 메뉴명, 수량, 단가, 소계
- "주문 확정" 버튼: 눈에 띄는 색상, 최소 44px 높이
- "뒤로가기": 장바구니 화면으로 복귀

### BR-ORDER-03: 주문 성공 처리
- 장바구니 자동 비우기 (localStorage 삭제)
- 성공 화면 표시: 주문 번호, 주문 시각, 총 금액
- 5초 후 메뉴 화면 자동 리다이렉트
- "메뉴로 돌아가기" 버튼으로 즉시 이동 가능

### BR-ORDER-04: 주문 실패 처리
- 장바구니 내용 유지 (삭제하지 않음)
- 에러 유형별 메시지 분기:
  - 네트워크 오류: "네트워크 연결을 확인해주세요"
  - 서버 오류 (5xx): "잠시 후 다시 시도해주세요"
  - 기타: "주문 처리 중 오류가 발생했습니다"
- "다시 시도" 버튼 제공
- 3회 이상 연속 실패 시 "직원에게 문의해주세요" 추가 안내
- 성공 시 실패 카운터 리셋

## 5. 주문 내역 규칙

### BR-HISTORY-01: 조회 범위
- 현재 활성 세션(ACTIVE)의 주문만 표시
- 이전 세션(COMPLETED) 주문은 표시하지 않음
- 시간 역순 정렬 (최신 주문 먼저)

### BR-HISTORY-02: 주문 상태 표시
- 대기중(PENDING): 노란색 배지
- 준비중(PREPARING): 파란색 배지
- 완료(COMPLETED): 초록색 배지
- 상태 텍스트: 한국어 표시

### BR-HISTORY-03: SSE 실시간 업데이트
- 주문 내역 화면 진입 시 SSE 연결 시작
- 화면 이탈 시 SSE 연결 해제
- 수신 이벤트:
  - ORDER_STATUS_CHANGED: 해당 주문 상태 즉시 업데이트
  - ORDER_DELETED: 해당 주문 목록에서 제거
- 연결 끊김 시 자동 재연결 (3초 간격, 최대 5회)
- 재연결 성공 시 전체 목록 재조회 (누락 방지)

### BR-HISTORY-04: 주문 상세 펼침
- 주문 항목 탭 시 상세 메뉴 목록 펼침/접힘 토글
- 펼침 시 표시: 메뉴명, 수량, 단가, 소계

## 6. UI/UX 규칙

### BR-UI-01: 터치 영역
- 모든 인터랙티브 요소: 최소 44x44px 터치 영역
- 버튼, 탭, 아이콘 모두 적용

### BR-UI-02: 가로 모드
- Landscape 고정 (1024x768 이상 기준)
- 사이드바 좌측 고정 (카테고리)
- 메인 영역 중앙 (메뉴 그리드)
- 장바구니 패널 우측 슬라이드

### BR-UI-03: 로딩 상태
- API 호출 중 적절한 로딩 인디케이터 표시
- 메뉴 목록: 스켈레톤 UI
- 주문 처리: "주문 처리 중..." 텍스트 + 스피너
- 자동 로그인: 로딩 인디케이터

### BR-UI-04: 에러 표시
- 에러 메시지는 사용자 친화적 한국어
- 내부 기술 정보 노출 금지
- 재시도 가능한 에러는 "다시 시도" 버튼 제공
- 토스트 메시지: 3초 후 자동 사라짐

### BR-UI-05: 금액 형식
- 원화(₩) 기호 사용
- 천 단위 구분자 (콤마)
- 예: ₩12,000, ₩0

### BR-UI-06: 접근성
- 모든 이미지에 alt 텍스트
- 색상만으로 정보 전달하지 않음 (상태 배지에 텍스트 병행)
- 충분한 색상 대비 (WCAG AA 기준)
- 포커스 표시 가시적
