# Business Rules - Unit 1: Common/Shared

## BR-01: 매장 코드 규칙
- 매장 코드는 영문 소문자, 숫자, 하이픈만 허용
- 최소 3자, 최대 50자
- 시스템 전체에서 유일해야 함
- 생성 후 변경 불가

## BR-02: 관리자 비밀번호 정책
- 최소 8자 이상
- 영문, 숫자 조합 필수
- bcrypt로 해싱하여 저장 (cost factor: 10)

## BR-03: 로그인 시도 제한
- 5회 연속 실패 시 계정 일시 잠금
- 잠금 시간: 15분
- 잠금 해제 후 실패 횟수 초기화
- 로그인 성공 시 실패 횟수 즉시 초기화

## BR-04: JWT 토큰 정책
- **관리자 토큰**: 유효기간 16시간
- **테이블 토큰**: 유효기간 365일 (장기, 태블릿 바인딩용)
- 토큰 페이로드: userId/tableId, storeId, role, exp
- 서명 알고리즘: HS256

## BR-05: 테이블 비밀번호 규칙
- 최소 4자리
- 숫자, 영문 허용
- bcrypt로 해싱하여 저장

## BR-06: 테이블 세션 규칙
- 테이블당 동시에 1개의 ACTIVE 세션만 존재 가능
- 첫 주문 생성 시 ACTIVE 세션이 없으면 자동 생성
- 이용 완료 시 세션 상태를 COMPLETED로 변경
- COMPLETED 세션의 주문은 고객에게 표시되지 않음

## BR-07: 주문 번호 생성 규칙
- 형식: `{매장코드}-{YYYYMMDD}-{순번4자리}`
- 예: `cafe-01-20260430-0001`
- 매장별, 일별 순번 리셋
- 유일성 보장

## BR-08: 주문 상태 전이 규칙
- 허용 전이: PENDING → PREPARING → COMPLETED
- 역방향 전이 불가 (COMPLETED → PREPARING 불가)
- PENDING → COMPLETED 직접 전이 불가 (반드시 PREPARING 거쳐야 함)
- 삭제는 모든 상태에서 가능 (관리자 직권)

## BR-09: 주문 금액 계산 규칙
- 주문 항목 소계 = 수량 × 주문 시점 단가
- 주문 총액 = 모든 항목 소계의 합
- 금액은 정수(원 단위)로 처리
- 음수 금액 불가

## BR-10: 주문 항목 스냅샷 규칙
- 주문 생성 시 메뉴명과 단가를 스냅샷으로 저장
- 이후 메뉴 가격이 변경되어도 기존 주문에 영향 없음
- 메뉴 삭제 시에도 기존 주문 항목은 유지

## BR-11: 이용 완료 처리 규칙
- 이용 완료 시 해당 세션의 모든 주문을 OrderHistory로 이동
- 이동 시 주문 항목을 JSON으로 직렬화하여 저장
- 원본 Order, OrderItem 레코드는 삭제
- 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋
- 이용 완료 시각 기록

## BR-12: 메뉴 노출 순서 규칙
- displayOrder 값이 작을수록 상위 노출
- 같은 카테고리 내에서만 순서 의미 있음
- 기본값 0 (새 메뉴는 맨 뒤에 추가)

## BR-13: 메뉴 가격 검증 규칙
- 최소 0원 (무료 메뉴 허용)
- 최대 10,000,000원
- 정수만 허용 (소수점 불가)

## BR-14: 이미지 업로드 규칙
- 허용 형식: JPEG, PNG, WebP
- 최대 파일 크기: 5MB
- S3 저장 경로: `{storeCode}/menus/{menuId}/{filename}`
- 업로드 성공 시 URL 반환

## BR-15: 데이터 격리 규칙
- 모든 데이터는 storeId로 격리
- 관리자는 자기 매장 데이터만 접근 가능
- 테이블 토큰은 해당 테이블의 주문만 접근 가능
- 다른 매장/테이블 데이터 접근 시 403 Forbidden

## BR-16: SSE 이벤트 규칙
- 이벤트 타입: ORDER_CREATED, ORDER_STATUS_CHANGED, ORDER_DELETED, TABLE_COMPLETED
- 매장별로 이벤트 스트림 분리
- 연결 타임아웃: 30분 (이후 재연결 필요)
- 재연결 시 누락 이벤트는 API 폴링으로 보완
